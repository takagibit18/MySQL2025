# 学生管理系统数据库 ER 图说明

## 一、数据库逻辑关系概述

本系统数据库包含 8 个核心业务表和 1 个扩展表（请假申请表），它们之间通过外键建立关联关系。考勤管理模块扩展了考勤表，并新增了请假申请表，实现了完整的考勤管理功能。

## 三、实体关系说明

### 1. 教师 (teachers) 与 班级 (classes) — 一对多关系
```
teachers (1) ──────────────────► classes (N)
   │                                │
   │  一个教师可以担任              │
   │  多个班级的班主任              │
   │                                │
   └── 工号 ◄───────────────────── 班主任工号 (FK)
```

### 2. 班级 (classes) 与 学生 (students) — 一对多关系
```
classes (1) ──────────────────► students (N)
   │                                │
   │  一个班级有多个学生             │
   │  一个学生只属于一个班级         │
   │                                │
   └── 班级ID ◄─────────────────── 班级ID (FK)
```

### 3. 教师 (teachers) 与 课程 (courses) — 一对多关系
```
teachers (1) ──────────────────► courses (N)
   │                                │
   │  一个教师可以教授多门课程        │
   │  一门课程只有一个授课教师        │
   │                                │
   └── 工号 ◄───────────────────── 授课教师工号 (FK)
```

### 4. 学生 (students) 与 课程 (courses) — 多对多关系
```
students (N) ◄────────────────► courses (N)
   │                                │
   │  一个学生可以选多门课程          │
   │  一门课程可以被多个学生选择      │
   │                                │
   └──────────► enrollments (选课表) ◄──────────
               (中间表，解决多对多关系)
```

### 5. 学生+课程 与 成绩 (grades) — 一对一关系
```
(学号 + 课程ID) ──────────────────► grades
   │                                  │
   │  一个学生的一门课程               │
   │  只有一条成绩记录                 │
```

### 6. 学生+课程 与 考勤 (attendances) — 一对多关系
```
(学号 + 课程ID) ──────────────────► attendances (N)
   │                                  │
   │  一个学生的一门课程               │
   │  可以有多条考勤记录               │
   │  (每次上课一条)                   │
```

### 7. 请假申请 (leave_requests) 与 考勤 (attendances) — 一对多关系（考勤模块扩展）
```
leave_requests (1) ──────────────► attendances (N)
   │                                  │
   │  一个请假申请可以关联            │
   │  多条考勤记录                    │
   │  (请假期间的每一天一条)          │
   │                                  │
   └── 请假申请ID ◄───────────────── 请假申请ID (FK)
```

### 8. 学生 (students) 与 请假申请 (leave_requests) — 一对多关系
```
students (1) ──────────────────► leave_requests (N)
   │                                │
   │  一个学生可以提交多个请假申请     │
   │                                │
   └── 学号 ◄───────────────────── 学号 (FK)
```

### 9. 课程 (courses) 与 请假申请 (leave_requests) — 一对多关系
```
courses (1) ──────────────────► leave_requests (N)
   │                                │
   │  一门课程可以有多个请假申请       │
   │                                │
   └── 课程ID ◄─────────────────── 课程ID (FK)
```

## 四、外键约束详情

| 子表 | 外键字段 | 父表 | 主键字段 | 删除规则 | 说明 |
|------|----------|------|----------|----------|------|
| students | 班级ID | classes | 班级ID | SET NULL | 删除班级时，学生班级ID置空 |
| classes | 班主任工号 | teachers | 工号 | SET NULL | 删除教师时，班级班主任置空 |
| courses | 授课教师工号 | teachers | 工号 | SET NULL | 删除教师时，课程教师置空 |
| enrollments | 学号 | students | 学号 | CASCADE | 删除学生时，级联删除选课记录 |
| enrollments | 课程ID | courses | 课程ID | CASCADE | 删除课程时，级联删除选课记录 |
| grades | 学号 | students | 学号 | CASCADE | 删除学生时，级联删除成绩记录 |
| grades | 课程ID | courses | 课程ID | CASCADE | 删除课程时，级联删除成绩记录 |
| grades | 录入教师工号 | teachers | 工号 | SET NULL | 删除教师时，成绩录入教师置空 |
| attendances | 学号 | students | 学号 | CASCADE | 删除学生时，级联删除考勤记录 |
| attendances | 课程ID | courses | 课程ID | CASCADE | 删除课程时，级联删除考勤记录 |
| attendances | 记录教师工号 | teachers | 工号 | SET NULL | 删除教师时，考勤记录教师置空 |
| attendances | 请假申请ID | leave_requests | 请假申请ID | SET NULL | 删除请假申请时，考勤记录请假申请ID置空 |
| leave_requests | 学号 | students | 学号 | CASCADE | 删除学生时，级联删除请假申请 |
| leave_requests | 课程ID | courses | 课程ID | CASCADE | 删除课程时，级联删除请假申请 |

### 删除规则说明：
- **CASCADE**：级联删除，删除父记录时自动删除子记录
- **SET NULL**：置空，删除父记录时将子记录的外键设为 NULL

### 核心关系：
1. **教师** 是系统的核心角色，关联班级、课程、成绩、考勤、请假审批
2. **学生** 属于班级，可以选课，有成绩和考勤记录，可以提交请假申请
3. **课程** 由教师教授，学生通过选课表关联
4. **选课表** 是学生和课程的桥梁（多对多关系的中间表）
5. **成绩表** 记录学生的学习成绩
6. **考勤表** 记录学生的出勤情况，可以关联请假申请
7. **请假申请表** 记录学生的请假申请，审批通过后自动创建考勤记录

## 七、考勤模块数据关系说明

### 考勤模块的核心关系链：

```
学生 (students)
  │
  ├─► 选课 (enrollments) ──► 课程 (courses)
  │                              │
  │                              └─► 教师 (teachers)
  │
  ├─► 考勤记录 (attendances) ──► 课程 (courses)
  │      │                          │
  │      └─► 请假申请 (leave_requests) ──► 课程 (courses)
  │              │                          │
  │              └──────────────────────────┘
  │
  └─► 请假申请 (leave_requests) ──► 课程 (courses)
```

### 考勤模块数据流转：

1. **点名流程**：
   - 教师选择课程和日期
   - 系统查询该课程的所有学生（通过enrollments表）
   - 教师进行点名，创建attendances记录
   - 考勤状态：1-出勤，2-迟到，3-早退，4-缺勤

2. **请假申请流程**：
   - 学生提交请假申请，创建leave_requests记录
   - 审批状态：0-待审批
   - 教师审批后，更新leave_requests的审批状态
   - 如果审批通过（状态=1），自动为请假期间的每一天创建attendances记录
   - 考勤状态：5-请假，并关联请假申请ID

3. **数据关联**：
   - attendances表的`请假申请ID`字段关联leave_requests表
   - 一条请假申请可以关联多条考勤记录（请假期间的每一天一条）
   - 考勤记录通过`学号`和`课程ID`关联学生和课程
   - 请假申请通过`学号`和`课程ID`关联学生和课程

## 八、索引设计

### 考勤表索引：
- 主键索引：`考勤ID`
- 普通索引：`学号`、`课程ID`、`考勤日期`、`请假申请ID`

### 请假申请表索引：
- 主键索引：`请假申请ID`
- 普通索引：`学号`、`课程ID`、`审批状态`、`(开始日期, 结束日期)`

索引设计优化了查询性能，特别是按学生、课程、日期、状态等条件查询时。
